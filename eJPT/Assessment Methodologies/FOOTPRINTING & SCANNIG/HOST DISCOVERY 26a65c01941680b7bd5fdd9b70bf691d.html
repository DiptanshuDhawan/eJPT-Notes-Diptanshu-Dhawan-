HOST DISCOVERY

PING SWEEPS :

  1. PING → ping { IP ADDRESS}
  1. NMAP → nmap -sn { IP ADDRESS }/ IDR // MORE RELIABLE AND EASY

A ping sweep is network scanning technique used to discover hosts within a specific IP address range on a network.

the basic idea is to send a series of ICMP Echo Request (ping) messages to a range of IP addresses and observe the responses to determine which addresses are inactive or reachable.

Don’t rely on ping sweeps for Windows systems as they often block ICMP packets and you’ll assume that windows hosts are offline but they were online.

NMAP SCANS :

  1. nmap -sn {TARGET IP ADDRESS OR RANGE} —> PING SCAN—> By default uses ICMP.
if we are there in the internal network than nmap will give priority to ARP scan to check for online hosts and not ICMP packets.

  1. nmap -sn -PS {IP} —> SYN PING // -sntells to skip port scanning and just find active hosts.
     SYN ping uses a TCP SYN packet to probe a target port, while ICMP ping uses an ICMP echo request.
    • SYN ping is often more reliable than ICMP ping.
    • Many firewalls block ICMP (common security measure).
    • TCP SYN packets (e.g., to ports 80/443) are usually allowed.
    • Hosts may ignore ICMP but respond to SYN pings.
    • Useful for host discovery in filtered networks.

BEST APPROACH TO CHECK IF HOST ALIVE OR NOT :

IF YOU HAVE A SINGLE IP:

  1. ping { IP } OR nmap -sn { IP }——> If doesn’t respond does not confirm dead!!

  1. nmap { IP } ——> run normal nmap scan to check if alive.
  1. nmap -Pn { IP } —> disable host discovery & force to scan ports even if host is down.

    SUCCESS!!! there are open ports which mean that host is also alive.

    In this case the host would be behind a firewall that filters ICMP traffic.
