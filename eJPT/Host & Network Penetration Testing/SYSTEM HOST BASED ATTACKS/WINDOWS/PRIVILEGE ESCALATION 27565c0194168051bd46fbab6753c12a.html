Bypassing UACwith UACMe—>

UAC [ User Account Control ] is a security feature that notifies you when a program tries to make changes requiring elevated privileges.

  • Admin Users: If you're an admin or part of the admin group, UAC asks for your consent to run programs with elevated privileges.
  • Standard Users: If you're not an admin, UAC asks for the admin password to allow the action.

  • UAC has various integrity levels ranging from lo to high , if the UAC protection level is set below high, windows programs can be executed with elevated privileges without prompting the user for confirmation.

  • There are multiple tools and techniques that can be used to bypass UAC, however the tool and technique used will depend on the version of the windows running on the target system.
  • USING UACMe to Bypass the UAC —> GITHUB: —> https://github.com/hfiref0x/UACME

  • AKAGAI…..this provides us with an executable that can be transferred on to the target system and that executable can be used to execute any payload that we want to execute with elevated privileges

SYNTAX —>

Run the executable from command line using the following syntax:

akagi32.exe [Method_Number] [Optional_Command]

OR

akagi64.exe [Method_Number] [Optional_Command]

Parameters:

  • Method_Number: Number corresponding to the UAC bypass method.{mentioned on GitHub}
  • Optional_Command: Full path to an executable file to run with elevated privileges
    • If omitted, the program will launch an elevated command prompt (%systemroot%\system32\cmd.exe)

Examples:

akagi32.exe 23
akagi64.exe 61
akagi32.exe 23 c:\windows\system32\calc.exe
akagi64.exe 61 c:\windows\system32\charmap.exe
