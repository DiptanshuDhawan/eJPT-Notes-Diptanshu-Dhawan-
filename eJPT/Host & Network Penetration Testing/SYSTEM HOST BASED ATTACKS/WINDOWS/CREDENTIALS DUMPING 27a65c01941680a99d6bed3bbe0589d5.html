CREDENTIALS DUMPING

Searching for PASSWORDS In WINDOWS CONFIGURATION FILES —>

  • Unattended files used for automated Windows installs (C:\Windows\Panther\Unattend.xml and C:\Windows\Panther\Autounattended.xml) can contain local account info including the Administrator password.
  • If left on the target after deployment, attackers can retrieve credentials (often stored as plain text or Base64-encoded) and authenticate to the system.
  • Mitigation: remove/secure these files immediately after installation (delete or move to secure storage), avoid embedding cleartext passwords in answer files, or use temporary credentials and change them post-install.

MIMIKATZ —>

  • What it is: Mimikatz — a Windows post‑exploitation tool that can extract credentials from Windows memory.
  • What it extracts: Cleartext passwords, NTLM hashes, Kerberos tickets and other credential material (when present in memory).
  • SAM database: The SAM (Security Accounts Manager) is the on‑disk Windows database that stores hashed local user passwords.
  • Where credentials live: Credential material is often cached in memory by LSASS (lsass.exe); tools can read these artifacts from LSASS memory.

• lsadump::sam: Extracts NTLM password hashes from the local Security Account Manager (SAM) database. Requires administrative privileges.


• lsadump::secrets: Dumps LSA secrets, which may include service account passwords and other system credentials.


• sekurlsa::logonpasswords: Retrieves plaintext passwords, NTLM, and LM hashes from the LSASS process for currently logged-on users. Requires debug privileges (privilege::debug).
