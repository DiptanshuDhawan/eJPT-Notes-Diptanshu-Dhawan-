NETWORK-BASED ATTACKS

SMB & NetBIOS Enumeration —>

  • NetBIOS and SMB are two different technologies, but they’re related in the context of networking and file sharing on Windows networks.

NetBIOS —>

(Network Basic Input/Output System)

  • NetBIOS is an API and a set of network protocols for providing communication services over a local network. It’s used primarily to allow applications on different computers to find and interact with each other on a network.
  • FUNCTIONS—> offers three primary services…
    • Name Service(NetBIOS-NS) : Allows computers to register, unregister and resolve names in local network.
    • Datagram Service (NetBIOS-DGM) : Supports connectionless communication and broadcasting.
    • Session Service (NetBIOS-SSN) : Supports connection-oriented communication for more reliable data transfers.
  • PORTS —>
    • 137 —> NAME SERVICE
    • 138 —> DATAGRAM SERVICE
    • 139 —> SESSION SERVICE

SMB—>

(Server Message Block)

  • SMBis a network file sharing protocol that allows computers on a network to shares files, printers, and other resources. It is the primary protocol used in Windows networks for these purposes.
  • FUNCTIONS —> provides features for file and printer sharing, named pipes, and inter-process communication(IPC). It allows users to access files on remote computers as if they were local.
  • VERSIONS —>
    • SMB 1.0: The original version which had security vulnerabilities, It was used with older operating systems like windows XP.
    • SMB 2.0/2.1: Introduced with Windows Vista/ Windows Server 2008, offering improved performance and security.
    • SMB 3.0+ : Introduced with Windows 8/ Windows Server 2012, adding features like encryption, multichannel support, and improvements for virtualization.
  • PORTS —> Generally uses port 445 for direct SMB traffic [bypassing NetBIOS] and port 139 when operating with NetBIOS.

  • While NetBIOS and SMB were once closely linked, modern networks rely primarily on SMB for file and printer sharing , often using DNS and other mechanism for name resolution instead of NetBIOS.
  • Modern implementations of Windows primarily use SMB and can work without NetBIOS, however, NetBIOS over TCP 139 is required for backward compatibility and are often enabled together.

smb-protocols.nse —> checks for SMB versions supported.

smb-enum-users.nse —> enumerate user accounts

SNMP—>

(Simple Network Management Protocol)

  • SNMP is a widely used protocol for monitoring and managing networked devices, such as routers, switches, printers, servers, and more.
  • VERSIONS —>
    • SNMPv1: The original version, using insecure plaintext community strings for authentication and offering no encryption or advanced error handling.
    • SNMPv2c : An improved version over v1 with faster bulk data retrieval (GETBULK) and better error codes, but still relies on the same insecure community string model.
    • SNMPv3: The secure standard, replacing community strings with a user-based model that provides strong authentication, data encryption, and access control.
  • PORTS—>
    • 161 UDP —> used for SNMP queries.
    • 162 UDP —> used for SNMP notifications

SNMP Enumeration —>

1. Nmap Commands

  • Detect SNMP service (UDP/161) + version detection:
    nmap -sU -p 161 -sV <target>
  • Run SNMP NSE scripts (gathers detailed SNMP info):
    nmap -sU -p 161 --script snmp-info,snmp-interfaces,snmp-processes <target>
  • Brute-force SNMP community strings (using a wordlist):
    nmap --script snmp-brute -p 161 <target>
  • Run all SNMP-related NSE scripts (use with caution):
    nmap -sU -p 161 --script "snmp-*" <target>
  • Scan multiple targets or subnets:
    nmap -sU -p 161 -T4 --script snmp-info <target1> <target2> <subnet-range>
  • Detect SNMPv3 (with specific username):
    nmap -sU -p 161 --script snmp-info --script-args snmpuser=<username> <target>
